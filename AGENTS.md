# AGENTS.md

## Project overview

Electron desktop app (TypeScript + React 17 + Redux Toolkit + Konva) for designing haptic experiences from audio. Three source roots under one `package.json`:

- `main/` — Electron main process, Webpack → CommonJS, Node target
- `frontend/` — Electron renderer (React), Webpack → browser target, dev server on :8080
- `shared/` — IPC channel names/types/wrappers; imported by both with no separate build

A Rust native addon (`native/bin/mac/{development,production}/HapticsSDK.node`) lives in a git submodule at `third-party/haptics-sdk`. macOS builds are **arm64-only** (no x86_64, no universal binary).

## Prerequisites

- **Node >= 24** (enforced in `engines`)
- **Yarn** (v1, lockfile present)
- **Rust stable toolchain** (for the native addon)
- macOS (CI is macOS-only; some companion/device tooling is macOS-specific)

First install: `yarn install` triggers `git submodule update --init --recursive` and builds the Rust addon via `scripts/build_native.js`.

## Development commands

| Task | Command |
|------|---------|
| Full dev app | `yarn start:dev` — starts frontend dev server + builds main + launches Electron |
| Lint main | `yarn main:lint` |
| Lint frontend | `yarn frontend:lint` |
| Typecheck main | `yarn main:typecheck` |
| Typecheck frontend | `yarn frontend:typecheck` |
| All unit tests (CI) | `yarn test:all` |
| Main tests (watch) | `yarn test:main` |
| Frontend tests (watch) | `yarn test:frontend` |
| Single test file (main) | `jest -c ./main/jest.config.js --runInBand path/to/test.ts` |
| Single test file (frontend) | `jest -c ./frontend/jest.config.js path/to/test.tsx` |
| E2E tests | `yarn test:e2e:prepare:mac` first, then `yarn test:e2e` |

## Lint, typecheck, test order

CI runs only `yarn test:all`. Lint and typecheck are separate manual steps. No pre-commit hooks. When making changes across both processes, run:

```
yarn main:lint && yarn frontend:lint && yarn main:typecheck && yarn frontend:typecheck && yarn test:all
```

## Architecture notes

### IPC is typed

All IPC channels in `shared/ipc-channels.ts` have corresponding payload types in `shared/ipc-types.ts`. Use `typedInvoke`, `typedSend`, `typedOn` from `shared/typed-ipc.ts` instead of raw `ipcRenderer`/`ipcMain` calls. Adding a new channel requires entries in both `ipc-channels.ts` and `ipc-types.ts`.

### Frontend state

Redux Toolkit store (`frontend/src/state/store.ts`) with 4 slices: `app`, `project`, `frame`, `editingTools`. Selectors in `frontend/src/state/selectors/`.

### Frontend editor

The waveform/effect editor uses react-konva (Konva canvas). Konva is mocked in tests.

### Native addon bridge

`main/src/hapticsSdk.ts` wraps the Rust NAPI library. Type definitions at `main/src/types/haptics_sdk_types.ts` are **auto-generated by `typeshare` from Rust code** — do not edit by hand.

### Auto-save

Projects auto-save every 500ms (`main/src/application.ts`).

## Testing specifics

- **Main tests** (`main/test/`): Node environment, heavily mocked (electron, mixpanel, dgram, uuid, logger, native modules). `HapticsSdkNapi` maps to the real `.node` binary. Must use `--runInBand` (socket/fs mocks are not parallel-safe).
- **Frontend tests** (`frontend/src/`): jsdom environment, `@testing-library/jest-dom`, `jest-canvas-mock`, CSS/SCSS stubbed, react-konva-utils mocked, react-dnd packages transformed.
- **E2E tests** (`e2e/`): Playwright with `electron-playwright-helpers`. Requires a pre-built & dir-packaged app. Run `yarn test:e2e:prepare:mac` first. Screenshot snapshots are darwin-specific. Only 1 worker (serial). Test timeout 20s.

## Lint conventions

- `no-console`: warns (`console.warn` and `console.error` allowed only)
- `prefer-const`: error
- Unused vars: prefixed with `_` to suppress
- `@typescript-eslint/no-explicit-any`: off
- Formatting rules disabled in ESLint (Prettier handles formatting, but no auto-run enforcement)
- `main/` has stricter checks than `frontend/` (frontend disables `no-unsafe-assignment`, `no-unsafe-call`, `no-unsafe-member-access`)

## CI

Single workflow (`.github/workflows/test.yml`): macOS, Node 24, Rust stable. Runs `yarn test:all` (main + frontend Jest). E2E tests and lint/typecheck are NOT part of CI.

## Packaging

Uses `electron-builder`. macOS builds are **arm64-only** (see `package.json` → `build.mac.target.arch`).

| Task | Command |
|------|---------|
| Build (arm64) | `yarn build:mac` — webpack main + frontend |
| Package (signed .dmg) | `yarn package:mac` or `yarn buildAndPackage:mac` |
| Dir package (unsigned .app) | `yarn dirpackage:mac` or `yarn buildAndDirpackage:mac` |
| Release | `yarn release:mac` |

The unsigned `.app` bundle at `dist/mac-arm64/Haptics Studio.app/` skips code signing entirely (`CSC_IDENTITY_AUTO_DISCOVERY=false`), useful for local iteration.

Windows variants (`:win` suffix) remain x64. The `main/webpack.config.electron.js` automatically picks `native/bin/{mac,win}/{development,production}/HapticsSDK.node`.
